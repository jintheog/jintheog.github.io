# 자바가 람다식을 편하게 쓰라고 미리 제공해놓은 표준 함수형 인터페이스들

- 람다식은 **“함수를 값처럼 넘긴다”** 는 개념이기 때문에 어떤 “형태”의 함수인지 표현하는 틀이 필요

| 함수형 인터페이스   | 매개변수 | 리턴값 | 의미                                       |
| ------------------- | -------- | ------ | ------------------------------------------ |
| `Runnable`          | 0        | 0      | 아무것도 안 받고 아무것도 안 돌려줌        |
| `Consumer<T>`       | 1        | 0      | 입력 1개 받고 소비(출력/저장)만 함         |
| `Supplier<T>`       | 0        | 1      | 아무것도 안 받고 값 하나를 만들어서 return |
| `Function<T,R>`     | 1        | 1      | 입력 1개 → 결과 1개                        |
| `BiFunction<T,U,R>` | 2        | 1      | 입력 2개 → 결과 1개                        |
